PSM Studio

Cannot Start PSM Studio

When PSM Studio startup fails, check the following:

If a version earlier than GTK#2.12 is installed in the development PC, PSM Studio may not start.

Workaround

Change the path C:Program Files (x86)GtkSharp2.12bin for the GTK# written in the environment variable PATH to an earlier version of the path.

Example:

Before

C:\Program Files\Common Files\Microsoft Shared\Windows Live;C:\Program Files (x86)\GtkSharp\1.00\bin;C:\Program Files (x86)\GtkSharp\2.12\bin

After

C:\Program Files\Common Files\Microsoft Shared\Windows Live;C:\Program Files (x86)\GtkSharp\2.12\bin;C:\Program Files (x86)\GtkSharp\1.00\bin;

Cannot Start PSM Studio on Windows XP

When PSM Studio does not startup on Windows XP, try the following.

  • Use Windows Update to update the OS to the latest version.
  • Install Microsoft Visual C++ 2008 redistributable package.
  • If Gtk# for .Net is not installed, install a versio of Microsoft .Net Framework 2.0 or later from the following URL and reinstall PSM SDK. http://www.microsoft.com/ja-jp/download/default.aspx

PSM Application Execution

Execution of the Application Fails on PlayStation(R)Vita

When application execution fails on PlayStation(R)Vita, the following dialog or error message will be displayed.

  • "An error has occurred. [hexadecimal error code]"

Likely contents of the error display are displayed below.

  • "Failed to connect the network." ... Connection to a network could not be established and access to the SCE authentication server failed.
  • "Failed to authenticate the Sony Entertainment Network." ... Account authentication failed.

Even though the Device Seed has been created, an error prompting recreation occurs

On an Android device supporting the multiuser feature, an error may occur prompting the creation of a key even though it is already created.

Because the Device Seed is created per user, if this error is displayed after the user was switched to the user starting the PSM application, please recreate the Device Seed.

Publishing Utility

Device Seed Creation Fails (Android)

If Device Seed creation fails with an Android device, check the following:

  • DevAssistant for Android is installed in the Android device.

Workaround

Execute %SCE_PSM_SDK%/tools/update_psmdevassistant.bat.

  • The Android device has not entered sleep mode. If the device has entered sleep mode, a connection with the development PC cannot be established.

Workaround

Change the settings so the Android device does not enter sleep mode.

If the conditions do not improve after performing the above, terminate the following processes in Windows Task Manager, then re-start them.

  • PsmDevice.exe
  • PsmStudio.exe
  • PublishingUtility.exe

Note: Windows Task Manager can be started with the Ctrl+Shift+Esc keys.

Device Seed Creation Fails (PlayStation(R)Vita)

If Device Seed creation fails with a　PlayStation(R)Vita, check the following:

  • PSM DevAssistant　for PlayStation(R)Vita is installed in the Vita.

If the conditions do not improve after performing the above, terminate the following processes in Windows Task Manager, then re-start them.

  • PsmDevice.exe
  • PsmStudio.exe
  • PublishingUtility.exe

App Key Creation Fails

If an App Key creation fails, check the following:

  • The Publisher Key registered on the server and the Publisher Key in the development PC are the same.

Workaround

Use Publishing Utility to recreate the Publisher Key.

By recreating the Publisher Key, the Publisher Key registered on the server will be updated at the same time, therefore the recreated key must be redistributed if a team is using the same Sony Entertainment Network account.
