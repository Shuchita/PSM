On Screen Controller

The On-screen Controller (hereafter, OSC) is an input device that uses the touch screen in place of a gamepad (physical controller). Part of the game screen is overlaid with a display of gamepad-style key buttons, and players touch them to perform operation. Developers can choose whether or not to use the OSC.

About OSC

To Use the OSC

To make this available, use Publishing Utility and set the GamePad item to True.

Refer to " Using Publishing Utility " - " Common Property" - "Input Device Settings" for the GamePad item of the Publishing Utility.

There are restrictions as described in the following "Compatible Environments" section.

Compatible Environments

The OSC can be used for Android devices.

On the other hand, it cannot be used for Simulator or PlayStation(R)Vita.

Compatibility Policy for Multiple Devices

The OSC will be compatible with the devices with which PSM is compatible, devices that have varying screen resolution/size and multiple screens. It is not scaled in its entirety to similar figures, the entire layout is adjusted using internal logic for button sizes and placements that are easy to use for players based on the device characteristics.

Player Customization

The Player Customization feature allows movement and size changing of each category by players (end users) using the OSC.

There are seven types of categories set by the OSC.

  • L button
  • R button
  • Left stick
  • Right stick
  • DPad Key (up, down, left, right key)
  • Face Button (triangle, circle, cross, square button)
  • Select/Start Buttons (includes Pull)

Opening the Player Customization Screen

After pressing the Android device's Menu button, select [Controller Settings] - [Screen Controller Settings], then tap the displayed [Wrench symbol] to open the Player Customization screen.

Customizable Items

Moving and size changing can be applied on categories.

  • Category moving

    By tapping and moving within the yellow category frame displayed on the screen, they can be moved to arbitrary positions.

  • Category size changing

    By tapping and moving the up/down and left/right buttons of the categories displayed on the screen, the size can be changed. Changes can be made within a range of half to twice the magnitude of initial values.

However, categories cannot overlap.

What cannot be performed with player customization operation

  • Buttons or categories cannot be hidden.
  • Customization cannot be performed on just the button itself and not the category.

Developer Customization

The OSC will be displayed within the screen, so it may overlap with the game screen and make the game hard to see. In order to improve such situations, developers can customize the OSC. This feature is called Developer Customization of the OSC.

  • Removing buttons
  • Adjusting the initial position of buttons: (can be readjusted by player customization)
  • Setting the OSC default display: (the player can be displayed/hidden using the menu)

Through this, buttons not needed for a game can be hidden, and buttons can be placed out of the way of important objects, etc. within the game screen.

This feature is optional. When this feature is not used, the full set for the OSC will be displayed in the standard position.

OSC Customize Tool

As mentioned earlier,

  1. Developers can change the position of buttons with Developer Customization. And
  2. The OSC internally adjusts the button layout in order to support a wide range of devices.

Since adjusting the former in consideration of the latter in order to achieve appropriate behavior on many devices is often difficult, a tool for supporting a range of operations is provided.

This tool is a GUI tool that operates in Windows. The items that can be changed with Developer Customization can be operated in a GUI using a mouse/numerical input. In addition, it is possible to confirm if the performed changes can be properly displayed on multiple devices with a device simulator.

osc.cfg

osc.cfg is the tool output file. The layout information created with the tool will be recorded to this file. Furthermore, osc.cfg describes with a text format called YAML, so a text editor can be used to open it, check its content, and directly text-edit it.

Layout Logic and Parameters

A list of parameters that can be confirmed in tabular form in the tool is recorded as-is to osc.cfg. The final layout is decided from these parameters using logic shared in the tool and target device.

The following is a simplified explanation of this layout logic. The layout logic may have its details adjusted and be updated to a newer logic. Here, we will explain the constant and fundamental rules that will not be affected by updates.

Placement will be changed based on a large-screen device (9 inches) that does not have many spatial restraints. The button and button category (including one or many buttons) sizes can be adjusted within this screen. At this time, the size is managed at actual size (units: inches). Meanwhile, the category position (vertical position) is specified with its proportions (percentage) within the screen in order to increase the affinity with the game screen.

With small devices, the proportions that the OSC comprises within the screen will be relatively large in order to maintain operability. From the layout created with the aforementioned procedure, if the device is small, the gap between categories is narrowed, categories are shifted, and finally the entire layout is reduced proportionately to fit it within the screen.

For further detailed rules, refer to the subsequent Layout Logic Details .

OSC Customize Tool

Overview

The OSC Customize Tool can perform movement and size changes for the OSC buttons.

By using the tool to add the output osc.cfg file to a PSM application, OSC can be customized.

Tutorial

  1. Start up the OSC Customize Tool
Execute %SCE_PSM_SDK%/tools/OscCustomizeTool/OscCustomizeTool.exe to start the OSC Customize Tool.
  1. Remove unnecessary buttons

By unchecking the check boxes in the Use tab of the Main Window, unnecessary buttons can be hidden. Here, try unchecking the "R Analog" and "DPad Key" check boxes.

  1. Layout adjustment procedure
Use the View Window and Main Window to edit the OSC placement information.

If you want to reset the procedure performed in the aforementioned editing, right-click on each category or button and select "Reset" displayed in the context menu.

The X, Y, Width, and Height in the table represent a category's horizontal position, vertical position, width, and height, respectively. The unit for Y is a percentage, units for others are inches.

  1. Changing the window size and checking the OSC placement

First, switch from Edit Mode to View Mode. Perform this switch using the [To View Mode] button at the upper left section of View Window. By operating the Slider Window and resizing the View Window with a mouse, the OSC placement can be confirmed at arbitrary screen sizes.

Move the slider portion of the Slider Window until it reaches four inches.

The window size of the View Window will change and the placement of the OSC on a four inch device can be confirmed.

At the same time, drag the borders of the View Window to perform a resize and make sure the placement has not failed within the range of the envisioned sizes.

  1. Switching to Dual Screen and adjusting

By selecting the Dual Screen tab displayed in the Main Window, the View Window display will switch to the corresponding screen.

Layouts for Dual are created in the same way as for Single. Each category and button can be edited. However, Dual is targeted to a limited number of devices, so it might be a good idea to use the standard settings as-is if you are not familiar with Dual devices.

  1. File output
From Menu in the Main Window, select [File (F)] > [Save (S)] and output the data file (osc.cfg) to the directory of the application that will use the OSC.
  1. Placement in the application directory
By adding osc.cfg directly below the project opened with PSM Studio and selecting [Build Action] - [Content] from the context menu, transfer to the device will be performed when the application is built. The filename must be osc.cfg.

Logic Version

The OSC Customize Tool calculates the display position of the OSC from the various parameters that have been specified, but the calculation results differ depending on the logic version. Unless there is a specific need to use an old version, we recommend using the latest version.

Tool Feature Reference and Menu Item Explanation

The OSC Customize Tool is composed of the following windows.

Main Window

The window to specify the various parameters related to the OSC with numerical values and check boxes, etc.

  • Menu

    • File

      Here, the osc.cfg file can be loaded and saved, etc.

      • New

        Initializes the OSC Customize Tool settings.

      • Load

        Loads the OSC Customize data file.

      • Save

        Saves the Customize data that is currently being edited to a file.

      • Exit

        Exits the tool.

    • Background

      Here, the settings can be performed for the background image displayed in the View Window. For example, a screen shot of the game screen can be displayed to compare balance with the OSC.

      • Load

        Loads an image file and displays it for the View Window background. For a Single image, the loaded image is displayed for the entire background. For a Dual image, it is displayed for the upper half.

      • Clear

        Clears an image loaded with Load in the Background menu.

  • Tabs

    • Use

      Specifies the display status for categories and buttons that will be used as the OSC.

      • Category

        Specifies the display status of each category.

      • DPad Key, Face Buttons, Select/Start

        Specifies the display properties of the buttons in the categories. If a parent category is hidden, all of the buttons in the category will be hidden.

      • Default Show

        Sets the OSC display at first startup (the player can be displayed/hidden using the menu).

        • ON: Display: (however, if a gamepad device exists in the Android device, it will be automatically set to hidden)
        • OFF: Hidden

    • Single Screen/Dual Screen

      Specifies the coordinates of each category and button.

      • Category

        Specifies the display coordinates of each category in inches. The origin is at the center of the rectangle. For the Y coordinate only, specification is a percentage from the top of the View Window client screen. In the case of a Dual Screen, specification is a percentage from the center (orange line) of the View Window client screen.

      • DPad Key, Face Buttons, Select/Start

        Button display coordinates are specified in relative coordinates from the center of the parent categories.

  • Other Items

    • Logic Version

      Specifies the logic version used for OSC placement.

    • System Bar

      This is a support feature that displays the System Bar in the View Window. The size of the System Bar on a device varies depending on the DPI (dots per inch) of the actual device. This feature should only be used as a guide.

View Window

This window displays the OSC using the parameters set in the Main Window. There are two types: Edit Mode for editing (by dragging the mouse), and View Mode for previewing.

  • Edit Mode

    Each OSC category and button position can be changed with a mouse. The window client size is fixed.

    • Button and category moving

      Each category and button can be moved arbitrarily, however they cannot be moved and their sizes cannot be changed outside of the set range.

  • View Mode

    Displays the OSC using the parameters changed in the Main Window and View Window Edit Mode. The window client size can be changed arbitrarily.

Slider Window

The View Window display size can be changed. The radio button located at the top indicates the View Window proportions, and the View Window size can be changed by moving the slider located at the bottom.

Other Specifications

Layout Logic Details

The adjustment features of the layout logic, which preferably should be understood for using the tool to customize the OSC, are introduced here.

  • Automatic category links

    Categories placed horizontally (with Y, the height being the same) upon designing, will be adjusted to the same height regardless of whether the layout logic is applied on a large or small device.

    If there is no link feature, a problem like the example below will occur. When using the L Analog Stick but not the R Analog Stick, the Face Button that is placed on the right column on a small device will be too high compared to the D-Pad Key and the layout will become unbalanced, as shown in the figure.

    Undesirable layout

    On the tool, a blue line will appear to align the categories when automatic links are enabled while dragging a category.

  • Padding

    When two categories are aligned vertically, appropriate padding will be inserted between the two to separate them.

Main Window Button Coordinates and the Possible Range of Movement in the View Window

When using a mouse to move each category and button in the View Window, movement outside the range set by the tool is prohibited.

  • L Trigger, L Analog, DPad Key

    Because the placement of this category is assumed to be on the left side of the screen, placement/movement to the right side of the screen is not possible. The X coordinate for the category is a relative coordinate (x>=0.0) from the left edge of the screen. Upon dragging on the tool, a red line will appear at approximately the middle of the screen. This means a category cannot be placed on the right of the line.

  • R Trigger, R Analog, Face Buttons

    Because the placement of this category is assumed to be on the right side of the screen, placement/movement to the left side of the screen is not possible. The X coordinate for the category is a relative coordinate (x<=0.0) from the right edge of the screen. Upon dragging on the tool, a red line will appear at approximately the middle of the screen. This means a category cannot be placed on the left of the line.

  • Select/Start

    A category can only be placed lower than categories other than Select/Start.

    The placement of the categories will be interpreted with the screen split into left/right/center. The placement will be interpreted as relative coordinates from the left edge of the screen (x>0.0) when placed on the left side, the right edge of the screen (x<0.0) when placed on the right side, and center placement when placed directly on the center (x=0.0).
