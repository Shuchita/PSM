PSM Studio

PSM Studioが起動できない

PSM Studioの起動に失敗する場合、以下の事項について確認してください。

開発用PCにGTK#2.12以前のバージョンがインストールされていると、PSM Studioが起動しないことがあります。

回避方法

環境変数PATHに記述されているGTK#へのパス C:Program Files (x86)GtkSharp2.12bin を以前のバージョンのパスより前に移動させてください。

例:

修正前

C:\Program Files\Common Files\Microsoft Shared\Windows Live;C:\Program Files (x86)\GtkSharp\1.00\bin;C:\Program Files (x86)\GtkSharp\2.12\bin

修正後

C:\Program Files\Common Files\Microsoft Shared\Windows Live;C:\Program Files (x86)\GtkSharp\2.12\bin;C:\Program Files (x86)\GtkSharp\1.00\bin;

Windows XPでPSM Studioが起動できない

Windows XPでPSM Studioが起動しない場合、以下の事項を試してください。

  • Windows UpdateでOSを最新の状態にする。
  • Microsoft Visual C++ 2008 再頒布パッケージをインストールする。
  • Gtk# for .Net がインストールできていない場合、 以下のURLから Microsoft .Net Framework 2.0 以上のバージョンをインストールし、 再度 PSM SDK をインストールする。 http://www.microsoft.com/ja-jp/download/default.aspx

PSMアプリの実行

PlayStation(R)Vita 上でのアプリケーションの実行に失敗する

PlayStation(R)Vita 上でのアプリケーションの実行に失敗した場合、以下のようなダイアログまたはエラーメッセージが表示されます。

  • "An error has occurred. [16進数エラーコード]"

主に発生しうるエラーダイアログの内容を以下に記載します。

  • "Failed to connect the network." ... ネットワーク接続が確立できず、SCEの認証サーバーへのアクセスに失敗した。
  • "Failed to authenticate the Sony Entertainment Network." ... アカウントの認証に失敗した。

デバイスシードを作成済みでも再作成を促すエラーが発生する

マルチユーザー機能に対応したAndroidデバイスの場合、既に鍵を作成しているにも関わらず鍵の作成を促すエラーが発生する事があります。

デバイスシードはユーザー単位で作成されるため、PSMアプリケーションを起動するユーザーに切り替えた後にこのエラーが表示された場合は、デバイスシードを再作成して下さい。

Publishing Utility

デバイスシードの作成に失敗する (Android)

Androidデバイスでデバイスシードの作成に失敗する場合、以下の事項について確認してください。

  • DevAssistant for AndroidがAndroidデバイスにインストールされていること。

回避方法

%SCE_PSM_SDK%/tools/update_psmdevassistant.bat を実行します。

  • Androidデバイスがスリープモードに移行していないこと。スリープモードに移行している場合、開発用PCとの接続が確立出来ません。

回避方法

Androidデバイスをスリープモードに移行しないように設定してください。

上記を実行しても症状が改善しない場合、Windows タスクマネージャから以下のプロセスを一度終了して、再度実行してください。

  • PsmDevice.exe
  • PsmStudio.exe
  • PublishingUtility.exe

※ Windows タスクマネージャは Ctrl+Shift+Escキーで起動できます。

デバイスシードの作成に失敗する (PlayStation(R)Vita)

PlayStation(R)Vitaでデバイスシードの作成に失敗する場合、以下の事項について確認してください。

  • PlayStation(R)Vita用PSM DevAssistantがVitaにインストール、および起動されていること。

上記を実行しても症状が改善しない場合、Windows タスクマネージャから以下のプロセスを一度終了して、再度実行してください。

  • PsmDevice.exe
  • PsmStudio.exe
  • PublishingUtility.exe

アプリ鍵の作成に失敗する

アプリ鍵の作成に失敗する場合、以下の事項について確認してください。

  • サーバーに登録されているパブリッシャ鍵と開発用PC上に配置しているパブリッシャ鍵が同一であること。

回避方法

Publishing Utility を用いて、パブリッシャ鍵を作成しなおしてください。

作成し直すことで、サーバーに登録されているパブリッシャ鍵も同時に更新されるため、 チームで同一のSony Entertainment Networkアカウントを使用している場合、作成しなおした鍵を再配布する必要があるので、ご注意ください。
