About creating and deleting threads

Detail

In PSM SDK 1.20.00, a problem is confirmed that some synchronization objects would leak when threads are frequently created and deleted on PS Vita and the program could hang up after about 420 times of repetitions.

Workaround

Please recycle a finite number of threads and avoid unlimited creation and deletion of threads.

About System.Net.WebConnectionStream

Detail

In PSM SDK 1.20.00, if your application uses System.Net.WebConnectionStream.Read() API and calls it repeatedly, your application may hang up after about 500 times calls. This is caused by the leak of some synchronization objects.

( WebConnectionStream class is usually retrieved as the return value of WebResponse.GetResponseStream() and so on. )

Workaround

Please reduce the number of call of Read() by enlarging the size of data which is read at one time. In addition, please enforce the garbage collection by calling GC.Collect() periodically.

About TwitterStreamingSample

Detail

In the TwitterStreamingSample, there is a problem that tweets are not shown even if you search.

In the TwitterStreamingSample, if time zone in the user's information settings of twitter account is not set correctly, there is a problem that program is forced termination.

We are investigating these issues now.
