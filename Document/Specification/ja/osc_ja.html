オンスクリーンコントローラ

On Screen Controller (以下 OSC) は、ゲームパッド（物理コントローラ）の代わりとして利用可能な、タッチスクリーンを用いた入力装置です。 ゲームの画面の一部に上書きしてゲームパッド風のキー・ボタンを表示させ、プレイヤーはそれらにタッチすることで操作を行います。 OSCを使うか使わないかは、開発者が選択することができます。

OSC について

OSC を使用するには

Publishing Utility を使用し、GamePad 項目を True にする事で利用可能になります。

Publishing Utility の GamePad 項目は、「 Publishing Utilityの使い方 」-「 Common Property 」-「入力デバイス設定」を御参照下さい。

以下の「対応環境」で述べる制限もあります。

対応環境

OSC は Androidデバイスにて使用できます。

シミュレータ、PlayStation(R)Vitaでは使用できません。

マルチデバイスへの対応方針

OSC は、PSM が対応する多様な画面解像度・サイズ、複数画面を持ったデバイスに対応します。 全体を相似形にスケーリングするのではなく、デバイス特性に応じて、プレイヤーの使いやすいボタンサイズ、配置になる様、内部のロジックで全体のレイアウトが調整されます。

プレイヤーカスタマイズ

プレイヤーカスタマイズ機能とは、OSCを利用するゲームプレーヤー(エンドユーザー)により、各カテゴリの移動やサイズ変更を行う事ができる機能です。

カテゴリはOSCが定める以下の７種類に分類されます。

  • Lボタン
  • Rボタン
  • 左スティック
  • 右スティック
  • DPad Key（上下左右キー）
  • Face Button（△○×□ボタン）
  • Select/Start ボタン（Pullを含む）

プレイヤーカスタマイズ画面を開く

AndroidデバイスのMenuボタンを押下後、[コントローラ設定] - [画面コントローラ設定]を選択し、[スパナマーク]が表示されているボタンをタップする事によりプレイヤーカスタマイズを開きます。

カスタマイズ可能な項目

カテゴリの 移動 と サイズ変更 が行えます。

  • カテゴリの移動方法

    画面に表示されている黄色のカテゴリ枠内をタップ＆ムーブする事により、任意の位置へ移動する事ができます。

  • カテゴリのサイズ変更方法

    画面に表示されているカテゴリの上下および左右ボタンをタップ＆ムーブする事により、サイズを変更する事ができます。 初期値の半分～２倍の範囲において変更できます。

ただし、他のカテゴリに重ねて置く事はできません。

プレーヤーカスタマイズで 出来ない事

  • ボタンやカテゴリを非表示にする事は出来ません。
  • カテゴリ単位ではない、ボタン単体に対してのカスタマイズは出来ません。

ディベロッパーカスタマイズ

OSC は画面内に表示されるので、ゲーム画面と重なりゲームが見にくくなる場合があります。 こういった状況を改善するために、開発者が OSC をカスタマイズする事ができます。 本機能をOSCのディベロッパーカスタマイズと呼びます。

  • ボタンを消す
  • ボタンの初期位置を調節する　：（プレイヤーカスタマイズにより再調整可能）
  • OSCのデフォルト表示状態を設定する　：（プレイヤーはメニューから表示/非表示を設定可能）

これにより、ゲームに不必要なボタンを表示しないようにし、またゲーム画面内の重要なオブジェクトなどを避けてボタンを配置したりする事ができます。

本機能はオプションです。本機能を使用しない場合は、フルセットの OSC が標準位置に表示されます。

OSCカスタマイズツール について

先に述べましたように、

  1. ディベロッパーカスタマイズにより開発者はボタンの位置を変更する事が出来ます。 また
  2. 広範囲のデバイスに対応する為、OSC は内部でボタンのレイアウトを調整します。

後者を考慮して前者の調整を行い、多様なデバイスで適切に振る舞う様に調整するのは通常困難であるため、一連の作業を補助するツールを提供いたします。

本ツールは、Windows 上で動作する GUI ツールです。 ディベロッパーカスタマイズで変更可能な項目を GUI 上からマウス・数値入力などにより操作できます。 また、行った変更が複数のデバイスで正しく表示できるかをデバイスシミュレータとして確認する事ができます。

osc.cfgについて

osc.cfg はツールの出力ファイルです。ツール上で作成したレイアウト情報が記録されています。 一方で、osc.cfg は YAML 形式というテキストで記述されていますので、テキストエディタで開き、内容を確認したり、または直接テキスト編集する事も可能になっています。

レイアウトロジックとパラメータについて

osc.cfg には、ツール上で表形式で確認可能なパラメータの一覧がそのまま記録されています。 ツール上、及びターゲットデバイス上で共通のロジックを用いてこのパラメータから最終的なレイアウトを決定します。

そのレイアウトロジックについて、簡単に説明します。 レイアウトロジックは、細部が調整され、より新しいロジックに更新される事があります。 ここでは更新に影響を受けない、普遍的かつ基礎的なルールを説明します。

配置の変更は、空間制約の少ない大画面デバイス（９インチ）を基準に行います。 この画面内で、ボタンおよびボタンカテゴリ（１～複数のボタンを含む）のサイズを調整します。 この時、サイズは実サイズ（単位はインチ）にて管理されます。 一方、カテゴリの位置（縦位置）については、ゲーム画面との親和性を高めるために、画面内での比率（パーセント）で指定します。

小型デバイスでは、操作性を確保するために、画面内にOSCが占める割合が相対的に大きくなります。 上記手順で作った基本レイアウトから、デバイスが小さい場合には、カテゴリ間の隙間を詰める、カテゴリをずらす、最後に全体を均質に縮小して、画面内に収めます。

これ以上の細かいルールについては、後述する レイアウトロジックの詳細 をご覧ください。

それでは次の章にて、実際にツールの操作を行いながら、OSC のディベロッパーカスタマイズをやってみましょう。

OSCカスタマイズツール

概要

OSCカスタマイズツールでは、OSCボタンに対して移動やサイズの変更をする事ができます。

ツールを使用して出力したosc.cfgファイルをPSMアプリケーションに追加する事により、カスタマイズされたOSCを利用する事ができます。

チュートリアル

  1. OSCカスタマイズツールの起動
%SCE_PSM_SDK%/tools/OscCustomizeTool/OscCustomizeTool.exeを実行し、OSCカスタマイズツールを起動します。
  1. 不要ボタンを消す

Main WindowのUseタブ内にあるチェックボックスをOFFにすると、不要なボタンを非表示にする事ができます。 ここでは、「R Analog」と「DPad Key」のチェックボックスをOFFにしてみましょう。

  1. レイアウトの調整作業
View WindowやMain Windowを利用し、OSCの配置情報を編集します。

3-1. ドラッグ

View WindowがEdit Modeの時、View Window内に表示されているボタンの移動やサイズ変更を、マウスを利用して行う事ができます。

View Windowに表示されているFace Buttonを編集してみましょう。 まず、△ボタンを○ボタンの上に移動させます。 次に、□ボタンを×ボタンの左に移動させます。

上記の編集で行った作業をリセットしたい場合は、各カテゴリやボタン上で右クリックし、コンテキストメニューに表示されるResetを選択して下さい。

3-2. 数値入力

Main Window内のSingle Screen / Dual Screenタブにある各種パラメータを編集することにより、View Windowに表示されるOSCの座標を編集する事ができます。

Single Screenタブ内にあるCategoryタブの、「L Analog」位置を編集します。 ここでは、「Face Button」とY座標や横幅、縦幅を揃えてみます。

表中の X, Y, Width, Height はカテゴリの横位置、縦位置、幅、高さを意味します。単位は、Yが100分率でそれ以外はインチです。

  1. ウィンドウサイズ変更してデバイス確認

最初に Edit Mode から View Mode に切り替えます。この切り替えは、View Window 左上の [To View Mode] ボタンにて行います。 Slider Windowの操作や、View Windowをマウスでリサイズする事により、任意の画面サイズでOSCの配置を確認する事ができます。

Slider WindowのSlider部分を操作し、4インチまで移動させます。

View WindowのWindowサイズが変わり、4インチデバイス上でのOSCの配置を確認する事ができます。

あわせて、View Window の境界をドラッグしリサイズを行い、想定される画面サイズの範囲内で、配置が破綻していない事を確認して下さい。

  1. デュアルに切り替えて調整

Main Windowに表示されているDual Screenのタブを選択する事により、View Windowの表示が対応した画面に切り替わります。

デュアルでもシングル同様にレイアウトを行います。各カテゴリやボタンの編集が可能です。 ただしデュアルは限られたデバイスを対象としているので、デュアルデバイスに詳しくない場合は標準設定をそのまま使うとよいでしょう。

  1. ファイル出力
Main WindowのMenuから、[File (F)] - [Save (S)]を選択し、OSCを利用するアプリケーションのディレクトリにデータファイル(osc.cfg)を出力します。
  1. アプリディレクトリに配置
PSM Studioで開いているプロジェクト直下にosc.cfgを追加し、コンテキストメニューから[ビルドアクション] - [コンテンツ]を選択することで、アプリケーションのビルド時にデバイスに転送されます。 ファイル名は osc.cfg でなくてはなりません。

ロジックバージョン

OSCカスタマイズツールは、指定された各種パラメータからOSCの表示位置を計算していますが、ロジックバージョンにより計算結果が異なります。 特別に過去のバージョンを利用する必要が無い場合は、最新のバージョンの利用を推奨します。

ツール機能リファレンス、メニュー項目解説

OSCカスタマイズツールは以下のウィンドウで構成されています。

Main Window

OSCに関連する各種パラメータを数値やチェックボックス等で指定するWindowです。

  • メニュー

    • File

      osc.cfgファイルの読み込みや書き込みなどを行う事ができます。

      • New

        OSCカスタマイズツールの設定値を初期化します。

      • Load

        OSCのCustomizeデータファイルを読み込みます。

      • Save

        現在編集中のCustomizeデータをファイルに書き込みます。

      • Exit

        Toolを終了します。

    • Background

      View Windowに表示する背景画像の設定を行う事ができます。 例えば、ゲーム画面のスクリーンショットを表示させて、OSCとのバランスを見たりする事ができます。

      • Load

        画像ファイルを読み込み、View Windowの背景に表示します。 読み込んだ画像はSingle画面の場合は背景全体に、Dual画面の場合は上半分に表示されます。

      • Clear

        BackgroundメニューのLoadで読み込んだ画像をクリアします。

  • タブ

    • Use

      OSCとして利用するカテゴリやボタンの表示状態を指定します。

      • Category

        各カテゴリの表示状態を指定します。

      • DPad Key, Face Button, Select/Start

        カテゴリに属するボタンの表示属性を指定します。 ただし、親となるCategoryが非表示の場合は、親に属するボタンは全て非表示になります。

      • Default Show

        初回起動時のOSC表示状態を設定します。（プレイヤーはメニューから表示/非表示を設定可能）

        • ON：表示　：（ただしAndroidデバイスにゲームパッドデバイスが存在する場合は、自動的に非表示設定になります）
        • OFF：非表示

    • Single Screen / Dual Screen

      各カテゴリやボタンの座標を指定します。

      • Category

        各カテゴリの表示座標をインチ単位で指定します。矩形の中心が原点です。 Y座標のみ、View Windowのクライアント画面上端からのパーセント指定になります。 Dual Screenの場合は、View Windowのクライアント画面の中央（橙線）からのパーセント指定になります。

      • DPad Key, Face Button, Select/Start

        親となるカテゴリの中心から、表示を行うボタンの座標を相対座標（インチ）で指定します。

  • その他の項目

    • Logic Version

      OSCの配置に使用するロジックのバージョンを指定します。

    • System Bar

      View WindowにSystem Barを表示する補助機能です。 実機のシステムバーのサイズはデバイスの DPI(dots per inch:ドット密度)によって変化します。本機能は参考程度にご利用ください。

View Window

Main Windowで指定したパラメータを使用し、OSCを表示するWindowです。 マウスドラッグによる編集用のEdit Modeと、プレビューを行うView Modeの２種類があります。

  • Edit Mode

    OSCの各カテゴリやボタンの位置をマウスにより変更する事ができます。 Windowのクライアントサイズは固定です。

    • ボタンやカテゴリの移動方法

      各カテゴリやボタンを任意に移動する事が可能ですが、定められた範囲以外への移動やサイズ変更は行う事ができません。

  • View Mode

    Main Windowや、View WindowのEdit Modeにより変更したパラメータを利用し、OSCを表示します。 Windowのクライアントサイズを任意に変更する事が可能です。

Slider Window

View Windowの表示サイズを変更する事ができます。 上部に配置されているラジオボタンはView Windowの比率となり、下部に配置されているスライダーを動かす事により、View Windowのサイズを変更する事が可能です。

その他の仕様

レイアウトロジックの詳細

ツールを使ってOSCをカスタマイズする上で理解していると好ましい、、レイアウトロジックの中の一部の調整機能を紹介します。

  • カテゴリの自動リンク

    デザイン時に横並びに配置されたカテゴリ（高さＹが同じ）は、大小デバイス上でレイアウトロジックを経ても同じ高さに調整されます。

    もしリンク機能が無かった場合には、例えば次の様な問題が起こります。 Lアナログスティックを使用するがRアナログスティックを使用しない時、小デバイス上で右列に配置される Face Button が D-Pad Key に比べて上に上がり過ぎて、図の様にアンバランスな配置になってしまう。

    アンバランスで期待しないレイアウト

    ツール上では、ドラッグ中に自動リンクが有効になった時にカテゴリを結ぶ青い横線が表示されます。

    カテゴリをドラッグして自動リンクが張られた図

  • パディング

    縦にカテゴリが二つ並ぶ時、小デバイス上では間に適切なパディングが入り、間が詰まりすぎない様に調整されます。

Main Windowのボタン座標と、View Windowでの移動可能な範囲

View Window上で、マウスにより各カテゴリやボタンを移動する際、ツールにより定められた範囲外への移動については制限されています。

  • L Trigger, L Analog, DPad Key

    画面の左側に配置される事を前提としているカテゴリなので、画面右側への配置・移動はできません。 カテゴリのX座標は画面左端からの相対座標(x>=0.0)になります。 Tool上でのドラッグ時には、およそ画面中央に赤線が表示されます。その線より右側への配置が出来ない事を表します。

  • R Trigger, R Analog, Face Button

    画面の右側に配置される事を前提としているカテゴリなので、画面左側への配置・移動はできません。 カテゴリのX座標は画面右端からの相対座標(x<=0.0)になります。 Tool上でのドラッグ時には、およそ画面中央に赤線が表示されます。その線より左側への配置が出来ない事を表します。

  • Select/Start

    Select/Start以外のカテゴリよりも下の座標にのみ、配置する事ができます。

    画面の左・右・中央に分割して解釈されます。左側に配置した場合は画面左端からの相対座標(x>0.0)。右側に配置した場合は画面右端からの相対座標(x<0.0)となり、厳密に中央(x=0.0)に配置した場合には、中央配置と解釈されます。
