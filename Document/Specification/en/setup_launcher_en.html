Setup for Executing with an Android Device

This document explains the settings for executing a PSM application on an Android device.

Note that this procedure requires an environment in which wireless connection can be made between an Android device and the network.

Preparing an Android Device

  1. The PSM application will be located on the SD card for an Android device - make sure the SD card is in place.
Note: Depending on the Android device, the SD card may be internal to the actual device. In such cases, an SD card does not have to be inserted.
  1. On the Android device, select [Settings] - [Application] - [Development], and check [USB debug].
  • If the device has Android version 4.X, place a check in [USB Debug] in [Settings] - [Developer Options].
  • If the device has Android version 4.2 or later, tap the area where [Build Number] is written in [Settings] - [Tablet Information/Device Information] seven times. When successful, place a check in [USB Debug] after [Developer Options] is displayed.

Note: The menu name will differ by Android device.

  1. Select [Settings] - [Date and Time] to automatically obtain the date/time from the network. When the date and time have been obtained, synchronize the date/time of the Android device with the development PC.
  2. Connect the development PC with the Android device using a USB cable.
  3. Install the device driver of the Android device to be used to the development PC.

Creating the App Key

Keys must be created to execute a PSM application on an actual device. The procedure for creating an App Key is explained as follows.

  1. Have preparation completed with Preparation for Executing PSM Applications on Actual Devices.
  1. Open the solution file of the PSM application to execute with the actual device on PSM Studio.
  2. The devices connected to the PC will be displayed on the drop-down list - select the target Android ( xxxxxx ).

Figure 2 Android Device Selection

Note: ( xxxxxx ) indicates the ID number of the Android device. The ID differs for each Android device.

  1. Select the menu's [Key] - [Update Device Seed and App Key] to create the device seed and App Key.

Figure 3 Device Seed and App Key Update

  1. Follow the instructions displayed in the dialog to make your input. The creation of the Publisher Key will be requested for the first time - create as instructed.

Note: If DevAssistant for Android is not installed in the Android device, DevAssistant for Android will be automatically installed in the Android device.

  1. Successful creation of the App Key completes the preparation process.

Execution

  1. After creating the App Key, rebuild the project with [Build] - [Rebuild All].
  2. Prepare the Android device for Internet connection. Upon starting up the PSM application, a connection will be made to the SCE server for authentication.
  3. From the menu, select [Run] - [Debug], or press the F5 key to transfer related files from the development PC to the Android device; the application will start up.
