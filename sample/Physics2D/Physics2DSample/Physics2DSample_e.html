Overview

This is a 2D physical simulation sample.

Collision processing and calculation of joints can be performed for basic shapes (box, circle, convex hull) and complex shapes (combination of basic shapes).

The basic usage is shown as samples.

Build of Physics2D engine library and samples.

Open Physics2DSample.sln and execute the build.

The library Sce.PlayStation.HighLevel.Physics2D.dll is inside SDK originally.

Build of only Physics2D engine library.

Open Physics2D.sln and execute the build to add changes and rebuild the library.

The reference to Sce.PlayStation.HighLevel.Physics2D.dll of Physics2DSample should be changed to the point at which is rebuilt, or use project reference in the place of assembly reference of Physics2D

Changes

Changes of this version are followings.
  • Added changes for Simulate API's parameter
  • Added new sample scenes
  • Added easy ways for setting rigid body and joint
  • Added simple raycast feature
  • Added spring feature
  • Added removement of joint and rigid body API, but those API may take relatively large calculation cost and should be careful
  • Internal calculation order was changed. Integrage,Broadphase,Narrowphase,Solver -> Broadphase,Narrowphase,Solver,Integrage
  • Because of calculation order change, Scene::UpdateFuncBeforeSim, Scene:UpdateFuncAfterSim, or sleep status of rigid bodies may change and may be required to rearrange a little
  • When you want to maintain same behaviours of physics objects completely as well as the previous version, please use Physics2D source code of the previous version
  • Few change to internal algorithm may change behaviours of physics objects, and it is recommended to save Physics2D source code before update of SDK version

Notice

Simulation becomes unstable for following cases.

Unstable for collision response

  • Simulation of the world of very small scale or large scale
  • Collision whose scale of rigid bodies (mass & length) are different too much
  • Change the default gravity and set it as very large value
  • Gravity of the mass is far from the center of geometry (the average of vertex positions)
  • Given vertices (as convex shape) which does not construct convex actually
  • Very tall stack of rigid bodies

Failure for collision detection

  • Cases that rigid body moves in high speed or large force given to rigid body may cause failure collision detection

Unstable for joint

  • Very heavy rigid body is linked to the joint
  • Two rigid bodies linked by the joint collide to each other for all the time
  • Very heavy rigid body collide with the object which is locked by the ball joint (fixed joint)

There are several cases whose simulation becomes unstable.

More detail information is mentioned inside each samples of TutorialScene. Please check those samples.
